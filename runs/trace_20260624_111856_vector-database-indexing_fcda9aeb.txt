## Background and Literature Review

### Hierarchical Navigable Small World (HNSW)

HNSW is a graph-based ANN indexing method that constructs a multi-layered navigable small world graph to facilitate efficient search [1]. The index organizes data points into hierarchical layers, with sparser connections at higher levels and denser connectivity at lower levels. Querying involves greedy graph traversal starting from the top layer, progressively refining candidate neighbors down to the base layer. This structure enables logarithmic search complexity and high recall rates.

HNSW supports dynamic insertions without requiring full index rebuilds, making it suitable for evolving datasets. However, the graph structure demands significant memory overhead—typically 2 to 5 times that of IVF-based methods—and incurs higher build times due to the complexity of graph construction and maintenance [2].

Recent academic advancements have introduced optimizations such as PCA-based filtering (pHNSW) to accelerate query throughput by up to 21x and hardware-aware designs (d-HNSW) that leverage RDMA and caching to reduce latency and energy consumption [3][4].

### Inverted File with Product Quantization (IVF-PQ)

IVF-PQ combines an inverted file system with product quantization to compress and index large-scale vector datasets efficiently [5]. The inverted file partitions the dataset into clusters (inverted lists), and product quantization encodes residual vectors within each cluster to reduce memory footprint. Querying involves identifying relevant clusters and performing approximate distance computations on compressed codes.

IVF-PQ is highly memory efficient and supports fast index construction, especially with GPU-accelerated implementations such as IVF-RaBitQ, which achieves up to 7.7x faster build times and 2.7x higher query throughput compared to baseline IVF-PQ [6]. However, standard IVF-PQ implementations typically yield lower recall (~89%) than HNSW, though recent optimizations have closed this gap to ~95% or higher [6].

IVF-PQ is best suited for billion-scale, mostly static datasets where memory constraints and build speed are paramount, but it generally requires full index rebuilds upon data updates.

## Background and Literature Review

### Approximate Nearest Neighbor Search

Exact nearest neighbor search in high-dimensional spaces suffers from the "curse of dimensionality," rendering brute-force methods computationally prohibitive at scale. ANN algorithms trade exactness for efficiency, returning neighbors that are approximately close with high probability.

### Hierarchical Navigable Small World (HNSW)

Introduced by Malkov and Yashunin (2018), HNSW builds a multi-layer graph where each layer is a navigable small-world network. The top layers contain fewer nodes, enabling long-range jumps, while lower layers provide finer granularity. Search proceeds by greedy traversal from the top layer down to the base layer, efficiently homing in on nearest neighbors [1].

HNSW is praised for:

- High recall without extensive parameter tuning.
- Low query latency due to graph traversal.
- Support for dynamic insertions without full rebuilds.

However, it requires substantial memory to store graph edges and has relatively high build times due to graph construction complexity [2].

### Inverted File with Product Quantization (IVF-PQ)

IVF-PQ combines two techniques:

- **Inverted File (IVF)**: The vector space is partitioned into Voronoi cells via clustering (e.g., k-means), and vectors are assigned to inverted lists.
- **Product Quantization (PQ)**: Each vector is compressed by splitting into sub-vectors and quantizing each independently, enabling compact storage and fast distance approximations [3].

IVF-PQ is widely used in large-scale systems due to:

- Lower memory footprint compared to graph-based methods.
- Faster index build times, especially with GPU acceleration.
- Good scalability to billions of vectors.

However, baseline IVF-PQ typically yields lower recall (~89%) than HNSW, requiring parameter tuning or enhancements to improve accuracy [4].

### Recent Advances

Recent research has focused on bridging the recall gap and improving efficiency:

- **GPU-native IVF-RaBitQ**: A GPU-accelerated IVF-PQ variant achieving 7.7x faster build times and recall improvements to ~95%+ [5].
- **PCA-based filtering in HNSW (pHNSW)**: Dimensionality reduction prior to graph traversal yields 14-21x throughput improvements [6].

These advances motivate a fresh comparative analysis incorporating state-of-the-art optimizations.

## Background and Literature Review

### Approximate Nearest Neighbor Search

Exact nearest neighbor search in high-dimensional spaces suffers from the "curse of dimensionality," rendering brute-force methods computationally prohibitive at scale. ANN algorithms trade exactness for efficiency, returning neighbors that are approximately close with high probability.

### Hierarchical Navigable Small World (HNSW)

Introduced by Malkov and Yashunin (2018), HNSW builds a multi-layer graph where each layer is a navigable small-world network. The top layers contain fewer nodes, enabling long-range jumps, while lower layers provide finer granularity. Search proceeds by greedy traversal from the top layer down to the base layer, efficiently homing in on nearest neighbors [1].

HNSW is praised for:

- High recall without extensive parameter tuning.
- Low query latency due to graph traversal.
- Support for dynamic insertions without full rebuilds.

However, it requires substantial memory to store graph edges and has relatively high build times due to graph construction complexity [2].

### Inverted File with Product Quantization (IVF-PQ)

IVF-PQ combines two techniques:

- **Inverted File (IVF)**: The vector space is partitioned into Voronoi cells via clustering (e.g., k-means), and vectors are assigned to inverted lists.
- **Product Quantization (PQ)**: Each vector is compressed by splitting into sub-vectors and quantizing each independently, enabling compact storage and fast distance approximations [3].

IVF-PQ is widely used in large-scale systems due to:

- Lower memory footprint compared to graph-based methods.
- Faster index build times, especially with GPU acceleration.
- Good scalability to billions of vectors.

However, baseline IVF-PQ typically yields lower recall (~89%) than HNSW, requiring parameter tuning or enhancements to improve accuracy [4].

### Recent Advances

Recent research has focused on bridging the recall gap and improving efficiency:

- **GPU-native IVF-RaBitQ**: A GPU-accelerated IVF-PQ variant achieving 7.7x faster build times and recall improvements to ~95%+ [5].
- **PCA-based filtering in HNSW (pHNSW)**: Dimensionality reduction prior to graph traversal yields 14-21x throughput improvements [6].

These advances motivate a fresh comparative analysis incorporating state-of-the-art optimizations.
