Understanding their trade-offs in recall, build cost, and query latency is critical for selecting appropriate solutions tailored to specific operational requirements.

## 1. Overview of Indexing Methods

### 1.1 Hierarchical Navigable Small World (HNSW)

HNSW constructs a multi-layered proximity graph where nodes represent vectors, and edges connect neighboring vectors. Search proceeds by navigating through these layers, starting from a high-level entry point and refining the search at lower levels. Key parameters include:

- **ef**: the size of the dynamic candidate list during search, influencing recall and latency.
- **M**: the maximum number of neighbors per node, affecting graph connectivity and memory.

**Advantages:**
- High recall (~95%) with default parameters.
- Supports dynamic insertions and deletions without rebuilding.
- Suitable for datasets up to approximately 10 million vectors with active updates.

**Limitations:**
- Higher memory consumption (2-5× that of IVF variants).
- Longer build times relative to static methods.

### 1.2 Inverted File with Product Quantization (IVF-PQ)

IVF-PQ partitions the vector space into clusters (inverted lists) and compresses vectors within each cluster using product quantization. Search involves probing a subset of clusters (controlled by **nprobe**) and performing approximate distance computations on compressed vectors.

**Advantages:**
- Efficient memory usage due to vector compression.
- Fast index construction, suitable for billion-scale static datasets.
- Lower build and update costs.

**Limitations:**
- Slightly lower recall (~89%) due to quantization errors.
- Recall depends on the **nprobe** parameter, which influences latency.
