## 1. Introduction

The unprecedented growth in large language models (LLMs) has necessitated architectural innovations that enable parameter scaling without a proportional increase in computational cost. Mixture-of-Experts (MoE) models address this challenge by routing each input token to a subset of specialized expert networks, thereby activating only a fraction of the total parameters per token. This conditional computation paradigm allows MoE models to achieve massive parameter counts while maintaining manageable inference costs [1,2,3].

Recent years have witnessed rapid evolution in MoE routing strategies and inference efficiency techniques. Early MoE models employed static Top-K routing, where a fixed number of experts are selected per token regardless of token complexity or task context. However, this approach can lead to suboptimal load balancing and inefficient resource utilization. More recent advances introduce adaptive routing methods, such as Ada-K Routing, which dynamically adjust the number of active experts per token to optimize the trade-off between computational cost and model performance [5]. Additionally, task-conditioned routing signatures reveal that routing decisions are structured and task-sensitive rather than random, enabling further routing optimizations [7].

Parallel to routing improvements, inference efficiency has been enhanced through system-level innovations. ExpertFlow, for example, integrates a transformer-based routing path predictor, token scheduling, and predictive expert caching to reduce GPU memory usage by up to 94% and increase inference throughput by up to 10× on single GPUs [6]. Complementary techniques such as expert pruning and skipping further streamline inference by eliminating redundant expert computations [8].

This report synthesizes these advances, providing a comprehensive overview of the state-of-the-art in MoE routing and inference efficiency from 2023 to 2025. We aim to clarify the technical developments, their implications for large-scale LLM deployment, and the challenges that remain.

## 2. Mixture-of-Experts Architectures: Background and Key Concepts

MoE models consist of a collection of expert subnetworks and a routing mechanism that assigns each input token to one or more experts. The routing function is critical, as it determines which experts are activated and thus directly impacts computational cost, model accuracy, and load balancing.

### 2.1 Static Top-K Routing

The foundational routing approach in MoE models is static Top-K routing, where the router selects a fixed number \( K \) of experts per token based on a learned gating function. For example, GShard employs top-2 routing, activating two experts per token, while Switch Transformers use top-1 routing [1,2,3]. Although simple and effective, static Top-K routing can lead to uneven expert utilization and suboptimal inference efficiency.

### 2.2 Adaptive Routing: Ada-K Routing

Ada-K Routing introduces a dynamic mechanism that adjusts the number of experts activated per token based on token-specific characteristics. This adaptive approach balances computational cost and model accuracy more effectively than static Top-K routing by allocating more experts to complex tokens and fewer to simpler ones [5]. Formally, the number of experts \( K_i \) for token \( i \) is determined by an adaptive gating function:

\[
K_i = f_{\text{adaptive}}(x_i)
\]

where \( x_i \) is the token representation. This flexibility improves load balancing and reduces unnecessary computation.

### 2.3 Task-Conditioned Routing Signatures

Recent research reveals that routing patterns are not random but structured and task-sensitive. Task-conditioned routing signatures analyze how routing decisions vary with different tasks, showing that the router implicitly learns task-specific expert utilization patterns [7]. This insight opens avenues for routing optimizations that leverage task context to improve efficiency and accuracy.

## 3. Advances in Inference Efficiency

Efficient inference in MoE models is challenging due to the need to manage expert loading, memory usage, and execution scheduling. Recent works propose several complementary techniques to address these challenges.

### 3.1 ExpertFlow: Predictive Expert Caching and Token Scheduling

ExpertFlow [6] presents a lightweight MoE inference system that combines:

- A transformer-based routing path predictor to anticipate expert activations,
- Token scheduling to optimize execution order,
- Predictive expert caching to reuse expert computations.

This integrated approach reduces GPU memory consumption by up to 94% and improves inference throughput by up to 10× on single GPUs, enabling more practical deployment of large MoE models.

### 3.2 Expert Pruning and Skipping

Post-training expert pruning and skipping methods identify and bypass experts that contribute minimally to model output, further reducing inference cost without significant accuracy loss [8]. These methods analyze expert importance and dynamically skip redundant computations, complementing routing-based efficiency gains.

## 4. Comparative Analysis of MoE Models (2023–2025)

Recent MoE LLMs such as GPT-OSS, DeepSeek-R1, LLaMA-4 Maverick, and Qwen3 differ in expert count, parameter scale, and routing strategies [1]. Table 1 summarizes key characteristics:

## 5. Challenges and Future Directions

Despite significant progress, several challenges remain:

- **Load Balancing:** Ensuring uniform expert utilization without sacrificing accuracy remains difficult, especially in adaptive routing schemes.
- **Routing Overhead:** The computational cost of complex routing functions can offset gains from conditional computation.
- **Scalability:** Efficiently scaling MoE models across distributed hardware while maintaining low latency is an ongoing engineering challenge.
- **Interpretability:** Understanding routing decisions and expert specialization is crucial for debugging and improving MoE models.

Future research should focus on developing lightweight, interpretable routing mechanisms, integrating routing with model training objectives, and exploring hardware-aware inference optimizations.

## 6. Conclusion

From 2023 to 2025, MoE large language models have advanced significantly through innovations in routing methods and inference efficiency. Adaptive routing schemes like Ada-K Routing and task-conditioned routing signatures enable more nuanced expert selection, improving load balancing and computational trade-offs. Concurrently, inference optimizations such as ExpertFlow’s predictive caching and expert pruning techniques dramatically reduce memory usage and increase throughput. Together, these advances position MoE architectures as a promising path toward scalable, efficient LLM deployment. Continued research addressing remaining challenges will be essential to fully realize their potential.
