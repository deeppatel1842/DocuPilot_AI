A. Concept and Definition of Agentic LLMs:

- Distinction: Traditional LLM = calculator; Agentic LLM = autonomous pilot.

B. Trend and Paradigm Shift:
- Industry shifted from scaling just model size to augmenting LLMs with external tools (APIs, functions) enabling extended capabilities.
- Tools have defined purpose, parameters, and outputs which the LLM calls autonomously to achieve tasks reliably.
- This paradigm represents a key evolution in AI agent design, emphasizing tool use and modular external functionality over raw model scale.
(Sources: Samira Ghodratnama 2025 overview, LinkedIn post 2025)

C. Frameworks and Methodologies:
- Agentic tool use research organizes into three main paradigms:
  1. Prompting as Plug-and-Play: guiding frozen LLMs to use tools through prompts and in-context signals without model updates.
  2. Supervised Tool Learning: training LLMs with labeled or synthetic data for robust tool-use behavior.
  3. Reward-Driven Tool Policy Learning: leveraging reinforcement learning to optimize long-horizon multi-step tool use via reward feedback.
- Each paradigm has distinct strengths and failure modes; combining insights is important for future agent design.
(Sources: arXiv preprint 2026)

D. Capabilities and Applications:
- Agentic LLMs autonomously plan, evaluate, self-correct, call tools (e.g., APIs, browsers, code generators), and coordinate with other agents.
- Used increasingly for tasks requiring multi-step reasoning, real-world interaction, and continuous decision making.

## 2. Background and Literature Review

### 2.1 Agentic Reasoning and Tool Use in LLMs

Agentic reasoning refers to the capacity of an LLM to autonomously plan, act, and learn within an environment, often by leveraging external tools or interacting with other agents. This paradigm contrasts with static prompt-based generation by emphasizing *agency*—the ability to make decisions, execute multi-step plans, and adapt based on feedback [1].

Recent scholarship has categorized agentic reasoning into three primary modes:

- **Single-agent reasoning:** An individual LLM plans and executes actions, often using internal reasoning and external tools.
- **Tool-based reasoning:** The LLM integrates external tools (e.g., search engines, calculators, APIs) into its reasoning loop to augment capabilities.
- **Multi-agent coordination:** Multiple LLM agents collaborate, communicate, and coordinate to solve complex tasks [1][2].

These modes reflect increasing complexity and capability, with tool use and multi-agent systems enabling richer interactions and problem-solving.

### 2.2 Planning in Agentic LLMs

Planning involves generating sequences of actions or queries to achieve a goal. In LLMs, planning can be realized through:

- **In-context reasoning:** Orchestrating plans via prompt engineering without model retraining.
- **Reinforcement learning and fine-tuning:** Post-training adaptation to improve planning policies.
- **Hierarchical planning:** Decomposing tasks into subgoals and subplans [2][3].

Effective planning frameworks balance flexibility, interpretability, and computational efficiency.

### 2.3 Reliability Challenges

Reliability in agentic LLMs encompasses correctness, robustness, and trustworthiness. Key challenges include:

- **Hallucinations:** Generating plausible but incorrect information.
- **Tool integration errors:** Failures in invoking or interpreting external tools.
- **Adaptation pitfalls:** Overfitting or catastrophic forgetting during continual learning.
- **Coordination failures:** Miscommunication in multi-agent systems [2][4].

Mitigating these issues requires feedback loops, memory mechanisms, self-evaluation, and domain-specific adaptations.
